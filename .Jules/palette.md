## 2024-05-17 - Added ARIA labels to audio player buttons\n**Learning:** Icon-only buttons used for play/pause and mute/unmute were missing proper ARIA labels, meaning screen reader users wouldn't know what the buttons did. Similarly, the PlayButton component calculated a title dynamically, but missed the aria-label attribute. Adding dynamic `aria-label` attributes ensures assistive technologies correctly announce the button state.\n**Action:** When creating icon-only buttons, always ensure an `aria-label` is present that accurately describes the button's action, and dynamically update it if the button's state changes (like Play vs Pause).
