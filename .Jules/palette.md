## 2024-05-27 - Keyboard Accessibility for List Items
**Learning:** React component lists utilizing `div` tags with `onClick` handlers fail basic accessibility standards because they cannot be triggered via the Enter or Space keys, and they lack proper ARIA roles and tab-focusability out of the box. Using semantic tags like `<button type="button">` immediately solves this by mapping to native browser accessibility APIs.
**Action:** Always favor native HTML interactive elements like `<button>` over styled `<div>` or `<span>` elements for clickable items. When converting blocks to buttons, ensure layout consistency is maintained by setting CSS properties like `w-full text-left`. Include visual indicators for keyboard focus like `focus-visible:ring-2` to accommodate keyboard users effectively.
